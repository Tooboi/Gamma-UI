Container

Containers are the building block of Gamma UI

They are responsive flex boxes that snap to breakpoints

Must use both container and container-{size} for responsive behavior

When used with a row you are set to add anything to your page

This paragraph is inside a container, the code below is inside a row

<div class="container container-md">
  <div class="row">
    ...content...
  </div>
</div>

Use the naming format: container-{size}

container will be 100% width under the set breakpoint

Otherwise it will snap to each breakpoint

When using with a navbar you should add a mt-5 class to compensate for the navbar height

Spacing

Margin & Padding

The classes are named using the format {property}{sides}-{size}

Property

  • m - to set margin
  • p - to set padding

Sides

  • t - set margin-top or padding-top
  • b - set margin-bottom or padding-bottom
  • s - set margin-start or padding-start
  • e - set margin-end or padding-end
  • x - sets both left and right
  • y - sets both top and bottom
  • blank - sets all four sides

Size

  • 0 - eliminate the margin or padding by setting it to 0
  • 1 - sets the margin or padding to 0.25rem
  • 2 - sets the margin or padding to 0.5rem
  • 3 - sets the margin or padding to 1rem
  • 4 - sets the margin or padding to 1.5rem
  • 5 - sets the margin or padding to 3rem
  • m-auto - sets the margin to auto

example: mx-5 p-0 mb-1

Color

Color is defined in the :root of the CSS you can change the variables and it will update throughout your app

  • col-bg - Background Color
  • col-surface - First Surface Layer
  • col-surface2 - Second Layer Up - Used for border and card fill body
  • col-surface3 - Lightest surface color used for a border on surface2
  • col-p-200 - Light primary color
  • col-p-500 - Rich mid-tone primary color
  • col-p-700 - Dark primary color
  • col-s-200 - Light secondary color
  • col-s-500 - Rich mid-tone secondary color
  • col-s-700 - Dark secondary color
  • col-on-surface - Text color on the surface like on a card
  • col-on-primary - Text color on primary colors like buttons
  • col-on-secondary - Text color on secondary colors like buttons
  • col-success - Success color for validation or buttons
  • col-danger - Danger color for validation or buttons
  • col-warning - Warning color for validation or buttons
  • col-info - Info color for focus objects
:root {
  --col-bg: #211c20;
  --col-surface: #3d353c;

Buttons

Button formatted with btn

The color and style classes are named using the format btn-{outline}-{color}

An outline variation of the button style can be created by adding the above outline tag

<button class="btn btn-primary">Primary</button>                  
<button class="btn btn-outline-primary">Primary</button>
